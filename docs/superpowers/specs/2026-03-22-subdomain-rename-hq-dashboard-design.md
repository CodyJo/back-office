# Subdomain Rename + HQ Dashboard

**Date:** 2026-03-22
**Status:** Approved
**Scope:** Rename admin.* subdomains to backoffice.*, add hq.codyjo.com cross-site dashboard

## Problem

Dashboards currently live at `admin.yourdomain.com`. The "admin" prefix is generic and could conflict with future admin tools. The project is called "Back Office" — the subdomain should match. Additionally, there is no cross-site portfolio view that shows comparative health, risk, and coverage across all managed sites.

## Two Changes

### 1. Subdomain Rename

All `admin.*` references become `backoffice.*`:

| Current | New |
|---------|-----|
| `admin.codyjo.com` | `backoffice.codyjo.com` |
| `admin.thenewbeautifulme.com` | `backoffice.thenewbeautifulme.com` |

**Files to change:**
- `config/backoffice.yaml` — `subdomain:` values in `dashboard_targets`
- `config/backoffice.example.yaml` — same
- `dashboard/site-branding.js` — hostname regex from `/^admin\.(.+)$/` to `/^backoffice\.(.+)$/`
- `backoffice/server.py` — ALLOWED_ORIGINS defaults
- `backoffice/api_server.py` — same
- `docs/LIVE-URLS.md` — documented URLs
- `docs/CICD-REFERENCE.md` — any admin.* references
- `terraform/cd.tf` — update references in comments/docs (not bucket names)

**What does NOT change:**
- S3 bucket names (internal identifiers, renaming is destructive)
- CloudFront distribution IDs
- DNS records (user's responsibility outside the codebase)

### 2. HQ Dashboard (`hq.codyjo.com`)

A new cross-site portfolio dashboard with three sections:

#### Health Scores

One card per site showing a composite 0-100 score. Weighted average of department scores:
- QA: 25%
- Compliance: 20%
- ADA: 20%
- SEO: 15%
- Privacy: 10%
- Product: 5%
- Monetization: 5%

Departments with no audit are excluded from the average (not counted as zero). Color-coded: green (80+), yellow (50-79), red (<50). Sorted best-to-worst.

#### Risk View

Table of findings per site sorted worst-first. Columns: site name, critical count, high count, medium count, low count, "Open in backoffice →" link. Row highlighting for sites with critical findings. Clicking a row filters the dashboard to that site.

#### Coverage Matrix

Grid with sites as rows, departments as columns. Each cell shows last audit date as relative time ("2d ago", "3w ago") with color-coded background:
- Green: <7 days
- Yellow: 7-30 days
- Red: >30 days
- Gray "—": never audited

#### Site Switcher Dropdown

At the top of the page. Default: "All Sites" (aggregated view). Selecting a site:
- Filters all three sections to that site's data in-place
- Sets `?site=name` query param
- Each site card/row also has an "Open in backoffice →" link to `backoffice.yourdomain.com`

"All Sites" resets to the aggregated view.

## New Data: `hq-data.json`

Generated by `backoffice/aggregate.py` during the aggregation step.

```json
{
  "generated_at": "2026-03-22T18:00:00Z",
  "sites": {
    "codyjo.com": {
      "health_score": 82,
      "department_scores": {
        "qa": 85, "seo": 90, "ada": 72, "compliance": 80,
        "privacy": 78, "monetization": 65, "product": 70
      },
      "risk": { "critical": 0, "high": 2, "medium": 8, "low": 15 },
      "coverage": {
        "qa": { "last_audit": "2026-03-21T10:00:00Z", "finding_count": 12 },
        "seo": { "last_audit": "2026-03-20T14:00:00Z", "finding_count": 5 },
        "ada": null,
        "compliance": { "last_audit": "2026-03-15T09:00:00Z", "finding_count": 3 }
      },
      "backoffice_url": "https://backoffice.codyjo.com"
    }
  },

    "health_score": 74,
    "risk": { "critical": 1, "high": 8, "medium": 22, "low": 45 },
    "sites_audited": 6,
    "sites_stale": 1
  }
}
```

**Health score formula:** Weighted average of available department scores per the weights above.

**Coverage data:** Extracted from each department's findings file `scanned_at` / `timestamp` / `metadata.auditDate` field (same extraction logic already in `backoffice/workflow.py:extract_scanned_at`).

**Backoffice URL:** Constructed from `org-data.json` product metadata or derived from target name: `https://backoffice.{domain}`.

## New File: `dashboard/hq.html`

Single HTML file following existing dashboard conventions:
- Dark theme, inline CSS, no build step
- Semantic HTML5, accessible color contrast
- Fetches `hq-data.json` on page load
- All interaction logic inline (no new JS files)
- Three sections: Health Scores (card grid), Risk View (table), Coverage Matrix (grid)
- Dropdown + `?site=name` query param for filtering

## New Dashboard Target

```yaml
# Added to config/backoffice.yaml deploy.aws.dashboard_targets:
- bucket: "hq-codyjo-site"
  base_path: ""
  distribution_id: ""           # set after CloudFront creation
  subdomain: "hq.codyjo.com"
  filter_repo: null              # aggregated — all sites
  allow_public_read: false
```

## Sync Manifest Changes

- Add `hq.html` to `DASHBOARD_FILES` in `backoffice/sync/manifest.py`
- Add `hq-data.json` to `SHARED_META_FILES` in `backoffice/sync/manifest.py`

## Aggregate Module Changes

Add `aggregate_hq()` function to `backoffice/aggregate.py` that:
1. Iterates all repos in results/ directory
2. For each repo, reads all department findings files
3. Extracts scores per department (reuses existing `extract_score` logic from workflow.py or equivalent)
4. Extracts `scanned_at` timestamps for coverage
5. Counts severities for risk view
6. Computes weighted health score
7. Writes `dashboard/hq-data.json`

Called from `aggregate()` main orchestrator alongside existing department aggregation.

## What's Untouched

- All agent scripts and prompts
- Per-site dashboard HTML files (qa.html, seo.html, etc.)
- Task queue, regression runner, delivery module
- Provider abstraction, sync engine logic
- Existing department-level aggregation
