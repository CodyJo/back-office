ADA-005
Form inputs missing associated labels
high Understandable WCAG 1.3.1 Level A AI Fixable
⚡
AI can fix this. The Back Office fix agent can add labels to these inputs automatically. make fix TARGET=analogify
Description
6 form inputs use placeholder text only with no <label> or aria-label. Screen readers announce these inputs as "edit text" with no context about what data is expected.
Evidence
<input placeholder="Your email"> <!-- no associated label --> <input placeholder="Gallery name"> <!-- no associated label --> <input placeholder="Client name"> <!-- no associated label -->
Impact
Screen reader users cannot understand what data each field expects. This affects an estimated 2-4% of web users who rely on assistive technology, and also breaks voice control software that uses label text for targeting.
File
📄 src/components/ContactForm.tsx:18
Recommended Fix
Add <label> elements with for/id association for each input. For inputs where a visible label isn't desired, use aria-label instead. Example:

<label for="email">Email</label>
<input id="email" placeholder="Your email">
